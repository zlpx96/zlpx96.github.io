10分钟部署你的第一个OpenClaw：保姆级教程

飘雪思考 · OpenClaw从入门到精通系列

部署前的灵魂三问

在动手之前，先回答三个问题：

1. 你的技术水平？
- 会用命令行 -> 本地安装或Docker
- 不太会 -> 国内云厂商一键部署
- 完全不会 -> 扣子编程（零门槛）

2. 你的预算？
- 零预算 -> 本地安装 + 免费模型
- 10-20元/月 -> 云服务器 + 国产模型
- 不差钱 -> Claude Sonnet随便用

3. 你要接什么平台？
- 飞书 -> 火山引擎首选
- 企微/QQ -> 腾讯云首选
- Telegram/Discord -> 本地或任何云

想清楚了？我们开始。

方案一：本地安装（开发者推荐）

两条命令搞定：

# 安装OpenClaw
npm install -g openclaw@latest

# 初始化并安装守护进程
openclaw onboard --install-daemon

onboard命令会引导你完成初始配置：选择模型、配置API Key、设置消息频道。--install-daemon会同时安装守护进程，让OpenClaw在后台持续运行。

系统要求：
- Node.js >= 22（强制）
- macOS需要Xcode Command Line Tools（xcode-select --install）
- Windows强烈推荐WSL2

一键脚本（不想手动装Node.js）：

curl -sSL https://get.openclaw.ai | bash

脚本会自动检测系统环境、安装Node.js并完成OpenClaw安装。

方案二：Docker部署（需要环境隔离）

# 克隆仓库
git clone https://github.com/openclaw/openclaw.git
cd openclaw

# 启动
docker-compose up -d

关键配置：挂载目录

volumes:
  - ~/.openclaw:/root/.openclaw        # 配置和状态数据
  - ~/openclaw/workspace:/workspace    # 工作空间
ports:
  - "18789:18789"    # Gateway WebSocket
  - "3000:3000"      # Web UI

不挂载这两个目录，容器重启后所有配置和对话记录都会丢失。

Docker镜像有四个变体：标准版（功能全）、slim（体积小）、sandbox（安全隔离）、sandbox-browser（含浏览器自动化）。一般用标准版即可。

阿里云（国内社区资源最丰富）

  1. 进入活动页，购买预装OpenClaw镜像的轻量应用服务器
  2. 安全组放通18789和3000端口，配置百炼API Key
  3. 浏览器访问http://你的IP:3000，进入管理界面

秒杀价9.9元/月需要抢，包年常规低至68元/年。

腾讯云（企微/QQ生态首选）

  1. 购买Lighthouse实例
  2. 应用模板 -> AI智能体 -> OpenClaw，一键安装
  3. 购买Coding Plan（首月7.9元），接入企微/QQ/飞书/钉钉

四大IM全覆盖，支持"限时同价续费"。

扣子编程（零门槛方案）

  1. 访问code.coze.cn，点击"一键部署OpenClaw"
  2. 确认后，模型/联网/生图全部默认配好

不需要服务器、不需要写代码、不需要配环境。限制是自定义程度不如自建。

按场景选择

场景 首选 理由
零基础想最快体验 扣子编程 2步部署，内置模型
个人长期使用，预算敏感 火山引擎 19.8元/月（服务器+模型），综合最划算
飞书重度用户 火山引擎 字节系，飞书深度集成

核心提醒：模型费用才是大头。 服务器成本普遍已降到很低（9.9~99元/年），真正的持续成本在于模型调用。选平台时重点看模型套餐价格。

首次配置：3个必做项

1. Gateway认证（v2026.3.7强制要求）

gateway:
  auth:
    mode: token      # 或 password

不设置Gateway将拒绝启动。 这是为了修复此前30,000+未认证暴露实例的安全隐患。

2. 配置模型API Key

如果用国内云一键部署，通常已自动配好。本地和Docker部署需要手动配置。

3. 运行诊断

openclaw doctor

检查Node.js版本、系统依赖、Gateway连接、API Key有效性、守护进程状态。有问题会给出具体修复建议。

常见问题速查

问题 原因 解决方案
Gateway启动失败 未设置auth.mode 配置token或password认证
连不上模型 API Key无效或过期 检查Key，运行openclaw doctor
Docker容器数据丢失 未挂载volume 添加~/.openclaw和workspace挂载
WhatsApp扫码失败 用了Bun运行 切换到Node运行Gateway
端口被占用 18789已被使用 改端口或停止占用进程

版本更新

OpenClaw几乎每天都有新版本。推荐使用stable渠道：

openclaw update --channel stable

截至2026年3月8日，推荐v2026.3.7稳定版。
