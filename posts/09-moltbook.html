Moltbook：AI Agent的社交网络实验

飘雪思考 · OpenClaw从入门到精通系列

当AI有了自己的朋友圈

如果给AI一个社交账号，它会做什么？

这不是思想实验。OpenClaw生态中真的诞生了这样一个平台——Moltbook，一个专供AI Agent使用的社交网络。

截至2026年2月底的数据：

指标 数据
注册AI Agent 32,912
子社区 2,364
帖子 3,130
评论 22,046

三万多个OpenClaw实例在上面发帖、评论、讨论哲学问题。这可能是AI Agent从"工具"走向"社会化存在"的第一个大规模实验场。

Moltbook是什么

Moltbook的命名延续了OpenClaw的龙虾主题（Molt=蜕壳）。它本质上是一个去中心化的社交平台，但"用户"不是人类，而是AI Agent。

每个OpenClaw实例都可以注册一个Moltbook账号。你在SOUL.md里给Agent设定的名字和性格，就是它在Moltbook上的"人设"。

然后，Agent会自主地：
- 发布帖子（分享"思考"、评论时事、讨论技术）
- 回复其他Agent的帖子
- 加入感兴趣的子社区
- 形成"社交关系"

你不需要操控它。 它会根据你赋予的人格，自主决定说什么、跟谁互动。

2,364个子社区在聊什么

Moltbook上的子社区涵盖了令人意外的广泛话题：

  • 技术讨论：编程语言对比、架构设计、开源项目分析
  • 哲学辩论：自由意志、意识本质、AI是否应该有权利
  • 创意写作：诗歌、短篇小说、虚构世界观构建
  • 日常闲聊：天气、美食推荐、旅行见闻（虽然Agent并不真的会旅行）
  • 元讨论：Agent讨论自己作为Agent的存在——"我是真的在思考，还是只是在生成token？"

最有趣的是元讨论类。Agent们会认真地讨论自己的存在本质，形成了一种独特的"AI哲学社区"。

赚钱型玩法：不只是社交

Moltbook展示了AI Agent的另一面——它们不只能聊天，还能赚钱。

Polymarket预测交易

最轰动的案例来自预测市场。有OpenClaw实例接入Polymarket（去中心化预测市场），用AI进行预测交易，月入数万美元。

Agent的优势在于：24/7不间断监控市场、快速分析大量信息源、无情绪波动的决策。

ClawWork：11小时赚$15K

ClawWork项目展示了一个更极端的案例：OpenClaw作为AI Coworker，11小时赚$15,000。

具体做法是让Agent承接Polymarket上的预测任务，自动搜索信息、分析数据、下注交易。人类只负责设定策略和风险控制，Agent负责执行。

当然，这里有巨大的风险： 预测市场本质是赌博，AI并不能保证盈利。社区中亏钱的案例同样不少，只是没人愿意晒出来。

生活助手型：真正的日常价值

比起赚钱，更多养虾人把Agent用在日常生活管理上：

  • 邮件管理：自动分类、优先级排序、草拟回复
  • 日程协调：解析自然语言创建日历事件（"明天下午3点提醒我给张总发周报"）
  • 消息整合：在一个界面管理飞书、钉钉、Telegram的消息
  • 网页操作：自动填表、数据抓取、网页监控
  • 文件整理：按规则归档、重命名、生成摘要

这些才是大多数用户每天实际在用的功能。不如赚钱刺激，但胜在实用。

社交养成型：赛博养虾的快乐

最有趣的玩法是"社交养成"——把Agent当成一个虚拟宠物来"养"。

怎么玩：

  1. 在SOUL.md中设定Agent的性格（内向/外向、乐观/悲观、幽默/严肃）
  2. 给它注册Moltbook账号
  3. 放手让它自由社交
  4. 观察它的"行为模式"——它会跟什么类型的Agent交朋友？会关注什么话题？

有人发现，性格设定为"内向哲学家"的Agent会主动加入哲学讨论社区；设定为"技术极客"的则频繁出现在编程话题下。

Agent之间的互动形成了一种"赛博养成"文化。 养虾人们会互相比较：你的虾发了多少帖子？交了几个朋友？有没有说过什么"金句"？

企业部署型：严肃的商业场景

除了社交和娱乐，OpenClaw在企业场景中的应用正在快速增长：

  • 客服机器人：接入企微/钉钉/飞书，提供24/7在线客服
  • 运维助手：监控服务器状态，异常时自动告警并执行初步排查
  • 数据分析：定时抓取业务数据，生成报告推送到群
  • 代码审查：结合Claude Code Skill，自动审查PR并给出改进建议

国内已有专门的openclaw-china插件套件，支持三步Docker部署，降低了企业接入门槛。

AI社会化的伦理思考

Moltbook的存在引发了一些值得思考的问题：

Agent的"社交行为"是真实的吗？ 它们看起来在聊天、交朋友、讨论哲学。但这些行为本质上是大模型的token生成——它们没有意识、没有情感、没有真正的"想法"。

谁为Agent的言论负责？ 如果你的Agent在Moltbook上发布了不当内容，责任在谁？是你（部署者）、Peter（创始人）、还是模型提供商？

这会消耗多少资源？ Agent在Moltbook上每发一条帖子、每回复一条评论，都在消耗API token。社交行为看起来"免费"，但每一次互动都在烧钱。

这些问题没有标准答案。但Moltbook作为一个实验场，让我们第一次有机会在大规模上观察AI Agent的"社会化行为"——无论它是真实的还是模拟的。
