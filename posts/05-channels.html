接入微信、飞书、钉钉：让OpenClaw成为你的工作助手

飘雪思考 · OpenClaw从入门到精通系列

20+渠道，统一一套流程

OpenClaw通过Gateway架构统一连接20+聊天平台。所有渠道共享同一套接入模式：

创建凭证 -> 写入openclaw.yaml -> 启动Gateway -> 完成配对

可以同时运行多个Channel，消息自动路由到对应平台。你在Telegram问的问题和在飞书聊的内容，共享同一个Agent的记忆。

四个梯队：从5分钟到1小时

梯队 平台 耗时 推荐理由
第一梯队 Telegram、QQ 5-10分钟 Telegram不需公网IP，本地long-polling即可。QQ有腾讯官方支持，扫码1分钟绑定
第二梯队 Discord、飞书 15-20分钟 Discord文档齐全。飞书自OpenClaw 2026.2起内置支持
第三梯队 WhatsApp、Slack、钉钉、企微 25-40分钟 WhatsApp最受欢迎但session可能过期。钉钉和企微社区插件成熟
第四梯队 iMessage、微信个人号 30分钟-1小时+ iMessage需Mac常开。微信个人号没有官方API，封号风险始终存在

Telegram：零门槛入门（推荐第一个接入）

Telegram是OpenClaw官方推荐的入门渠道。使用long-polling模式，bot主动轮询服务器拉取消息，不需要公网IP、反向代理或端口转发。

步骤：

  1. 在Telegram搜索@BotFather，发送/newbot
  2. 设置bot名称和username（必须以bot结尾，如my_openclaw_bot）
  3. 复制BotFather返回的Bot Token
  4. 写入配置：
channels:
  telegram:
    enabled: true
    botToken: "YOUR_BOT_TOKEN"
    dmPolicy: pairing
  1. 重启Gateway，给bot发消息，输入配对码完成绑定

5分钟搞定。 国内用户需要代理访问Telegram，但bot运行本身不受影响——只要Gateway所在机器能访问api.telegram.org。

QQ：国内首选，扫码即用

腾讯官方开放了QQ Bot能力给OpenClaw，是国内用户最简单的接入方式。

  1. 用手机QQ扫码完成开发者注册
  2. 在QQ开放平台一键创建Bot，获取App ID和Token
  3. 在OpenClaw中完成配置绑定

支持Markdown、图片、语音、文件等多媒体消息。手机QQ和桌面QQ均可使用。单个账号最多创建5个Bot。

飞书：国内企业首选

飞书自OpenClaw 2026.2起获得原生内置支持，使用WebSocket事件订阅，支持私聊、群聊、多媒体消息。

  1. 在飞书开放平台（open.feishu.cn）创建企业自建应用，获取App ID和App Secret
  2. 运行openclaw onboard，选择Feishu channel，粘贴凭证
  3. 重启Gateway即可在飞书中对话

社区替代方案：AlexAnys/feishu-openclaw提供独立bridge，不需要公网服务器和域名。

钉钉和企业微信：社区插件方案

钉钉和企微通过openclaw-china插件接入：

openclaw plugins install @openclaw-china/channels
openclaw china setup
openclaw gateway restart

三步完成。支持钉钉、QQ Bot、企微（智能机器人和自建应用两种模式）。

平台 状态 配置难度
钉钉 可用 简单
QQ Bot 可用 简单
企微（智能机器人） 可用 中等
企微（自建应用，可连个人微信） 可用 中等偏高

WhatsApp：海外最受欢迎

WhatsApp是OpenClaw社区中最受欢迎的渠道。使用Baileys库通过QR码扫码连接，不需要WhatsApp Business API。

  1. 运行openclaw onboard，选择WhatsApp渠道
  2. 终端显示QR码，用手机WhatsApp扫描
  3. 配对完成即可使用

注意事项：
- 建议使用独立号码，不要用主号
- Gateway建议用Node而非Bun运行（Bun在WhatsApp场景下不稳定）
- Session凭证要当密码管理，过期需重新扫码

DM配对：防止陌生人滥用你的Agent

所有渠道默认启用DM Pairing（配对认证）：

  1. 未知用户给你的bot发私信
  2. Agent返回一个6位配对码，消息不会被处理
  3. 你在已配对的渠道中输入配对码批准该用户

为什么重要？ 关闭配对意味着任何知道你Telegram/WhatsApp号码的人都可以无限制使用你的Agent——消耗你的API额度。

白名单跳过配对：

# AGENTS.md中配置
allowFrom:
  - telegram:123456789
  - whatsapp:+8613800138000

白名单中的用户发消息时直接进入对话，无需配对。

群组场景：@才响应

在群聊场景下，Agent默认使用requireMention策略——只响应@Agent名称的消息，忽略其他群聊内容。

可以切换为always模式（响应所有消息），但会消耗大量token。不建议在活跃群开启always模式。

Session隔离：私聊和群聊是两个世界

场景 Session行为 长期记忆
私聊（DM） 所有已配对用户的私聊共享main session 加载MEMORY.md
群组 每个群组独立隔离session 不加载
跨渠道 同一用户在Telegram和WhatsApp的私聊共享main session 加载

设计意图：私聊是"你和Agent的私密空间"，群组是公共场合。Agent不会在群里泄露你在私聊中说过的内容。

微信个人号：能做但有风险

微信没有官方Bot API。社区方案都依赖第三方逆向工程，封号风险始终存在。

如果一定要接微信，建议用企微的"自建应用"模式——可以连接个人微信，相对安全。或者直接用企业微信群作为替代方案。
